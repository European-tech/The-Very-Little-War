## CRITICAL Findings

### FINDING-CODE-PAGE-001: CRITICAL - No Auth Guard on messageCommun.php Allows Mass Message Injection

**File:** `/home/guortates/TVLW/The-Very-Little-War/messageCommun.php:1-9`

**Description:** This file has no authentication guard, no session check, no CSRF protection, and no authorization check. It inserts a hardcoded message to every player in the database. Any anonymous visitor who discovers this URL can trigger mass message insertion into every player's inbox.

**Code:**
```php
<?php
include("includes/connexion.php");
$ex = dbQuery($base, 'SELECT login FROM membre');
$timestamp = time();
while($d = mysqli_fetch_array($ex)){
    dbExecute($base, 'INSERT INTO messages VALUES(default, ?, ...)', 'is', $timestamp, $d['login']);
}
```

**Fix:** Either delete this file entirely (it appears to be a one-time admin script from 2015), or add admin authentication, CSRF protection, and make it POST-only. The `ecriremessage.php` file already supports `[all]` mass messaging for the admin account "Guortates", making this file redundant.

### FINDING-CODE-PAGE-010: HIGH - Double Escaping in ecriremessage.php

**File:** `/home/guortates/TVLW/The-Very-Little-War/ecriremessage.php:9-11`

**Description:** POST data is escaped with `mysqli_real_escape_string()` before being used in prepared statements via `dbExecute()`. Since prepared statements already handle escaping, this causes double-escaping. Users will see backslashes before quotes in their messages.

**Code:**
```php
$_POST['titre'] = mysqli_real_escape_string($base, antihtml($_POST['titre']));
$_POST['destinataire'] = ucfirst(mysqli_real_escape_string($base, $_POST['destinataire']));
$_POST['contenu'] = mysqli_real_escape_string($base, $_POST['contenu']);
```

**Fix:** Remove all `mysqli_real_escape_string()` calls since `dbExecute()` uses prepared statements:
```php
$_POST['titre'] = antihtml($_POST['titre']);
$_POST['destinataire'] = ucfirst($_POST['destinataire']);
// $_POST['contenu'] needs antihtml() too for XSS prevention
$_POST['contenu'] = antihtml($_POST['contenu']);
```
